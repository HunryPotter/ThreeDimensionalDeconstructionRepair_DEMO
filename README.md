# 三维结构修理原型 (-DEMO)

本项目是一个高科技 HMI 风格的三维结构修理交互原型系统。旨在为工程师提供直观的损伤标记、结构检索及技术请求处理工作流。

## 1. 核心工程架构

项目基于模块化组件设计，各核心功能区拆分为独立 Feature 模块：

```text
/src/features
  ├── spatial-view/         # 三维渲染核心、交互时序组件 (Timeline) 与全局 UI 弹窗
  ├── record-sidebar/       # 多维结构修理看板 (模块化重构)
  │   ├── components/       # 视图子组件 (SelectionView, DrillDownView, AtaTreeView)
  │   ├── RecordSidebar.js  # 核心协调层 (Coordinator)
  │   └── RecordSidebar.css # 样式隔离文件
  ├── external-case/        # CASE 业务系统集成与专业标点工作流
  ├── navigation-bar/       # 顶部导航与全局状态
  ├── detail-sidebar/       # 右侧业务数据详情面板
  └── components/           # 全局公用组件 (PopupManager 等)
```

## 2. 关键功能模块

### 🕒 三维损伤时序看板 (3D Timeline)
- **多维度视图**：支持 SR (结构修理) 与 CRS (适航指令) 的时序切换。
- **智能过滤**：具备自适应时间轴缩放、日期区间预设（如 2026 Q1）及自定义日期检索。
- **星标联动**：选中的损伤记录在时间轴上以动态星标（Star Icon）高亮，并与侧边栏及 3D 模型实现双向同步。

### 🌳 多维侧边栏检索系统 (Multi-tab Sidebar)
- **双引擎树结构**：
    - **机身部段 (Section)**：基于飞机物理位置的层级索引。
    - **ATA 章节 (ATA)**：基于民航标准章节的结构索引，支持各级多选框（Checkbox）及其级联勾选逻辑。
- **高级过滤组件**：集成型别、航司、架次的多选下拉框，支持面包屑导航与关键字模糊搜索。
- **数据导出功能**：支持将当前过滤后的 ATA 结构树及损伤条目一键导出为 **Excel (CSV)** 格式，内置 UTF-8 BOM 以兼容中文环境。
- **高度集成 HMI**：全站弃用浏览器原生弹窗，改用自研的 **“冰川白”毛玻璃质感 UI**。
- **智能业务约束**：针对新增标点实施严格的 ATA 树节点选中逻辑，确保数据归属的工程严谨度。
- **实时状态同步**：实现 Popups、侧边栏与 3D 场景的深度联动，删除/更新数据后全站 UI 零延迟同步。

### ⚡ CASE 技术请求与标点模式
- **全流程闭环**：支持从维修记录发起技术请求，并跳转至 CASE 系统进行三维位置补录。
- **专业标点体系**：
    - 独立的轻量化 3D 标点环境与 **内部 UI 确认工作流**。
    - **树枝强制校验**：必须选中 ATA 结构树的分支节点方可进行三维标点。
    - **持久化预览**：标点确认后即时同步至全局数据流，支持 Popup 信息框的同步清空与隐藏。

## 3. 技术设计规范

- **交互美学**：遵循工业级 HMI 设计，全站使用 `Inter` 字体、几何对齐布局与动态模糊（Backdrop Filter）动效。
- **模块化架构**：采用 **Coordinator 模式** 拆分复杂 UI 逻辑，确保 `RecordSidebar` 等核心模块的高可扩展性。
- **性能优化**：通过事件驱动（Event-Driven）架构解耦组件，确保大规模数据渲染时的响应速度。
- **自适应布局**：侧边栏支持动态高度伸缩与滚动位置记忆，优化长列表操作体验。

## 4. 项目预览与操作指南

如果您是第一次拿到此工程代码，请按照以下步骤进行环境搭建与功能预览：

### 环境准备
- **Node.js**: 推荐 v16.x 或更高版本。
- **浏览器**: 推荐使用最新版的 Chrome 或 Edge 浏览器以获得最佳 3D 渲染效果。

### 快速启动
```bash
# 1. 安装依赖
npm install

# 2. 启动本地开发服务器
npm run dev
```
启动后，访问浏览器显示的本地地址（通常为 `http://localhost:3000`）。

### 核心功能体验路径 (Golden Path)

为了快速了解本原型的核心能力，建议按以下顺序进行操作：

1.  **进入二级视图**：
    *   在首页列表中点击任意一行记录 -> 点击右下角 **“确认”** 按钮。
2.  **体验双索引结构树**：
    *   在左侧边栏切换 **“机身部段”** 与 **“ATA 章节”** 标签。
    *   展开 ATA 章节，尝试点击 **多选框**，体验级联勾选逻辑。
    *   使用顶部的筛选器（航司、架次等）观察时间轴和列表的实时联动。
    *   点击侧边栏标题右上角的 **📥 (导出)** 图标，获取当前筛选后的数据报表。
3.  **时序联动与三维交互**：
    *   在底部时间轴上点击圆形标记（Marker），观察模型上的热点自动弹出详情。
    *   点击右上角的 **“恢复视图”** (🏠) 重置 3D 视角。
4.  **CASE 系统完整闭环**：
    *   在左侧列表点击一条损伤记录 -> 点击详情页底部的 **“发起技术请求” (⚡)**。
    *   在弹出的技术参数页点击 **“CASE 系统”** 进入管理面板。
    *   点击右侧面板的 **“添加三维损伤标记”** 进入标记模式。
    *   在模型上任意位置点击，确认坐标并保存。
    *   **验证点**：确认后蓝色标点会保留在视图中；点击删除标记后，标点会同步消失。
